사용자 여정

첫 방문 → 홈(캔버스) 빈 상태 온보딩 → 에셋/관계 추가
↓
시뮬레이션 실행 → 트리거 에셋 선택 → 결과 확인
↓
모니터링 대시보드 → 상태 테이블 + 알림 이력
↓
설정 (고급) → 에셋/관계 테이블 CRUD

Step 1: 글로벌 CSS 정리

servers/frontend/src/app/styles/index.css 에 Vite 스캐폴딩 잔여물이 남아있다:

#root { max-width: 1280px; text-align: center; } — 캔버스 전체 높이에 방해

body { place-items: center; } — 센터 정렬 제거 필요

.logo, .card, .read-the-docs — 미사용 클래스

이를 앱에 맞게 정리하고, #root를 전체 뷰포트를 채우도록 변경한다.

servers/frontend/src/app/layout/AppLayout.css의 .app-main에서 캔버스 페이지일 때 padding 없이 전체 높이를 채우도록 조정한다.

Step 2: 캔버스 홈 + 빈 상태 온보딩

servers/frontend/src/pages/canvas/ui/AssetsCanvasPage.tsx 수정:

에셋 0개일 때 ReactFlow 위에 오버레이 표시:

"Factory MES에 오신 것을 환영합니다" 제목

"에셋 추가" 버튼 (기존 setAddFormOpen(true) 활용)

간단한 사용 가이드 텍스트 (3줄 정도)

기존 toolbar, graph, panels 동작은 그대로 유지

Step 3: MonitoringPage 생성

새 파일: pages/monitoring/ui/MonitoringPage.tsx, .css, index.ts

기존 AssetList.tsx 기반으로 확장:

상태 요약 카드: 정상/경고/에러 건수 표시

Asset+State 테이블: 기존과 동일 구조

자동 새로고침: useEffect에 30초 polling interval 추가 + 수동 새로고침 버튼

알림 이력 섹션: getAlerts(20) REST API 호출하여 최근 알림 테이블 표시 (기존 미사용 API 활용)

Step 4: SimulationPage 생성

새 파일: pages/simulation/ui/SimulationPage.tsx, .css, index.ts

기존 AssetsPage.tsx 의 시뮬레이션 섹션(line 168-513)에서 추출:

트리거 에셋 드롭다운: getAssets()로 목록 로드, <select>로 선택 (기존 assets[0] 하드코딩 제거)

maxDepth 입력: 기본값 3, 숫자 input

실행 버튼: one-shot 실행 / 지속 실행 시작 / 중단

결과 이벤트 테이블: 기존과 동일

에셋이 없으면 "먼저 에셋을 추가하세요" + 홈으로 이동 링크

Step 5: SettingsPage 생성

새 파일: pages/settings/ui/SettingsPage.tsx, .css, index.ts

기존 AssetsPage.tsx (시뮬레이션 제외) + RelationshipsPage.tsx를 하나의 페이지로 통합:

탭 전환: "에셋" | "관계" 두 탭

에셋 탭: 에셋 목록 테이블 + 생성 폼 + 인라인 편집 (기존 AssetsPage에서 시뮬레이션 섹션만 제거)

관계 탭: 관계 목록 테이블 + 생성 폼 + 인라인 편집 (기존 RelationshipsPage 그대로)

Step 6: 라우트 + 네비게이션 재편

servers/frontend/src/app/routes.tsx 변경:

{ index: true, element: <AssetsCanvasPage /> }
{ path: 'monitoring', element: <MonitoringPage /> }
{ path: 'simulation', element: <SimulationPage /> }
{ path: 'settings', element: <SettingsPage /> }

기존 /assets, /relationships, /canvas → 제거 (또는 redirect)

servers/frontend/src/app/layout/AppLayout.tsx 네비 변경:

홈 (/) | 모니터링 (/monitoring) | 시뮬레이션 (/simulation) | 설정 (/settings)

servers/frontend/src/app/layout/AppLayout.css 에서 .app-main이 캔버스(홈)일 때 padding 없이 flex-grow 되도록 조정.

Step 7: 테스트 유틸 및 기존 테스트 업데이트

servers/frontend/src/test/utils.tsx의 AppWithRoutes를 새 라우트 구조로 업데이트.

기존 페이지 테스트 파일에서 import 경로 변경이 필요한 경우 수정.

변경하지 않는 것

기존 entities/ 레이어 (API, types, AlertToast 등) — 그대로 유지

기존 shared/ 레이어 — 그대로 유지

AppLayout의 SSE 알림 토스트 구독 — 그대로 유지

기존 pages/home/, pages/assets/, pages/relationships/ 파일 — 삭제하지 않고, 라우트에서만 분리 (필요시 참조용)
