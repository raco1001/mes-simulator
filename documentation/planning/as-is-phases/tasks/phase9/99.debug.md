P0-1. Backend Kafka 환경변수 추가

파일: docker/docker-compose.app.yml

mes-backend 서비스에 Kafka 연결 설정이 없어서 컨테이너 내부에서 localhost:9092(기본값)로 연결 시도, 실패한다.

추가할 환경변수:

Kafka**BootstrapServers: 'kafka:9093'
Kafka**TopicAssetEvents: 'factory.asset.events'
Kafka**TopicAlertEvents: 'factory.asset.alert'
Kafka**ConsumerGroupId: 'backend-alert-consumer'

P0-2. Scoped IAlertStore -> Singleton BackgroundService DI 수명 충돌 수정

파일: servers/backend/DotnetEngine/Infrastructure/Kafka/KafkaAlertConsumerService.cs

현재 KafkaAlertConsumerService(BackgroundService = singleton)가 생성자에서 IAlertStore(scoped)를 직접 주입받는다. .NET의 Development 환경에서는 scope validation이 활성화되어 있어 \*\*InvalidOperationException: Cannot consume scoped service from singleton 에러로 앱 시작이 실패한다.

수정 방법: 생성자에서 IServiceScopeFactory를 주입받고, 메시지 처리 시 scope를 생성하여 IAlertStore를 resolve한다.

// Before: IAlertStore alertStore 직접 주입
// After:
private readonly IServiceScopeFactory \_scopeFactory;

public KafkaAlertConsumerService(
IServiceScopeFactory scopeFactory, // scoped 대신 scope factory
IAlertNotifier alertNotifier, // singleton이므로 그대로
IOptions<KafkaOptions> options,
ILogger<KafkaAlertConsumerService> logger) { ... }

// ProcessMessageAsync 내부:
using var scope = \_scopeFactory.CreateScope();
var alertStore = scope.ServiceProvider.GetRequiredService<IAlertStore>();
alertStore.Add(alert);

P1-1. docker-compose.full.yml 삭제

파일: docker/docker-compose.full.yml

infra.yml과 사실상 동일한 내용이며, 앱 서비스 섹션이 비어있다. MongoDB 버전 불일치(mongo:7.0 vs infra.yml의 mongo:8)까지 존재하므로 삭제한다. infra.yml + app.yml 조합으로 충분하다.
